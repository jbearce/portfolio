German Sentence Generator (C++)

This is a project I had been thinking about making for a long while. Back in high school, I realized that it would be helpful to have completely random exercises, where the student would need to insert the correct word into a sentence. To achieve this, I considered building a program that outputs the same sentnece, in both English and German, but replaces one German word with a blank. So far, my project successfully generates sentences in both English and German. At some point, I plan to revisit this to expand its features, including more complex sentence structures and more streamlined code. The ideal end result would be a framework that could be easily modified to accept a wide number of languages.

I started out by building a functional command line application. Once that was complete, I used the code as a base for a new GUI version, using Qt Creator to build a simple 2-Window interface. The main window simply shows the two generated sentences side by side, with a button to generate a new word, and a button to open the word lists in a new window.

The Qt integration was trickier than I expected, not because Qt was unexpectedly challenging, but because I had structured the code somewhat naively. I was compiling just the main file, and including all files into that in a cascade. Thanks to this roadblock, I gained first hand experience into why robust project structures are useful. To complete the Qt conversion, I modified my code structure to only use includes for header files. Other files are instead compiled into the project as dependencies, which Qt Creator appears to handle automatically.

If you'd like, you can check out both versions in my GitHub repository. For the CLI version, check the "develop" or "master" branches. For the GUI version, check the "gui-version" branch.

Experimenting With NCurses & ASCII Art Games

I thought it would be fun to work on one of the simplest computer games I could think of—a game map with obstacles, stored in a 2D array, with a literal "character" that the user can move around the screen.

When I started this project, I set the goal of using the terminal as my game screen. Since I couldn't find much on how to properly reset the screen using the standard I/O stream, I dove into NCurses. NCurses takes over the terminal window, does stuff, then hands control back over to the I/O stream when finished. While sensible in retrospect, it took me a bit of experimentation to figure out how this sytem functions, and the help of a few generous developers whose forum posts I stumbled across.

If you'd like, you can check out this project in my GitHub repository.
